Social Network Aggregationa SWP12 project proposal

Social Network Aggregation: Problem Description

  • Social Networks overtook email as No.1 communication in 2009.

  • email is still No.1 in business communication.

  • There is no integration between these two communication paradigms.

Integrate email into Social Networks

  • Solution I

    • Send and receive all mails with your Social Network client.
    • Integrate existing email addresses by allowing Social Network providers to access your mail accounts.

  • Drawbacks

    • There is more than one Social Network your contacts use.
    • Do you really want to give your mail pass to StudiVZ?

Integrate Social Networks into email

  • Solution II (preferred)

    • Allow for aggregation of Social Network activities of your friends
    • Access activity streams with different Social Network APIs
    • Save your Social Network credentials in your mail client
    • Render activity stream right beside your mail documents

  • Drawbacks

    • There is more than one Social Network your contacts use.
    • There exists more than one mail client

The Semantic Web

... in one slide.

a web of data that can be processed directly and indirectly by machines
  • An extension to the existing World Wide Web
  • Adds a layer of meaning by using shared vocabularies
  • Linked Data instead of linked documents
  • Can be seen as a distributed database of global scale

Goals

With Semantic Web / Linked Data Technologies, we can achieve …

  1. Portability through de-centrally shared open vocabularies.
  2. Identity though Uniform Resource Identifier (URI/IRI) and the architecture of the WWW.
  3. Linkability as a consequence of 2) as well as 1).
  4. Privacy though transport layer security and de-centralization.

Semantic Filesharing

an (incomplete) feature list:

  • Allow users to upload files and share it with DSSN users
  • Allow users to talk / discuss files
  • Provide activity streams of these discussions based on atom feeds
  • Provide a semantic pingback service to allow back-linking
  • Integrate all these components very well

Technical Considerations

Comprehensive preliminary (PHP) projects:

  • data.fm - a data hub with WebID support
  • OntoWiki - a Semantic Data wiki incl. extensive API Erfurt
  • lib-dssn - an API for creating and consuming activity streams

There are similar projects available for other technology stacks, so PHP is not mandatory.
